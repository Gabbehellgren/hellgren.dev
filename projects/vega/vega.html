3.2 What is a VPN?

A VPN or Virtual Encrypted Network, is a sort of "fake" network between the user and server. This fake network is call ed a tunnel. The tunnel is private, this means that the tunnel is only used between a user/group of users and the server. It is also encrypted and this means that the packages or "messeges" are distorted in a specific way and then sent, on recive the message can be decrypted aka the message is returned to it original. Encrytion is good because you have to have the right "key", "pattern" or "instructions" to decrypt and read it, this means that people that are not ment to access the package can not read it because it is just a distorted mess.
Earlier I said that VPN is a fake network, to explain you can think of your phone. You search www.hellgren.dev this becomes a request sent to your router (a computer that handles requests and sends information to the right device) the router then give you the data you requested (searched). It is the same way with VPN:

The user requests some data → the request is encrypted → sent to the server → the server decrypts → the server gets the data → encrypts → send to user → user decrypts → the user got the data.

3.4 Why I choosed WireGuard

When I began my "research" I started by trying to set up OpenVPN but it proved to be a hard and slow configuration. I then did som searching and found that there is many VPN protocols like OpenVPN, IPSec and WireGuard. I looked in to the options and found that WireGuard unlike the others is really fast, still secure and yet a very simple protocol. So every fact talked for WireGuard:

  1. Fast
  2. Secure
  3. Easy and fast configuration

3.5 What is WireGuard?

WireGuard is a modern, fast, and secure VPN protocol that is designed to be simpler than traditional solutions like OpenVPN or IPSec. It uses state-of-the-art cryptography and is built directly into the Linux kernel, which makes it lightweight and efficient. Unlike older VPNs, WireGuard is easy to configure and provides high performance with minimal overhead, making it a popular choice for both personal and enterprise use. Go to wireguard.com↗ for more information.

3.6 Why build my own software?

I choosed to build my own software because I wanted to be able to share my VPN with my friends. But I wanted to control when and how long my "clients" used my VPN. I want to control because I do not want others to surf on my personal network. That might have me responsible for what they do on internet. I want to be able to share my VPN without comprimising myself. It would also look sleeker way to share the VPN and it would mean that I do not have to share my public IP as much.
